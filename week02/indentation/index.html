Why should we indent?

For a number of reasons!

  1. It makes it easier to see which child elements are nested in which parent elements.
  2. It makes your code easier to read, and therefor easier to understand, especially for someone else who might be looking at your code in order to help you.
  3. It makes it easier to catch bugs and other errors
  4. If your code isn't indented correctly, and you ask for help, I'm going to ask you to indent your code before I help you ;) (in my experience, just going back through your code to clean it up often reveals your error to you anyway!)

Should I indent every nested element?

For the most part! Sometimes, like if you are writing an H1 or H2 or H3 tag, those tags can go on the same line, so you don't have to indent the content of those tags on their own line.

For example:

This is fine to have on one line:

<h5>This is an H5 element</h5>

Some things to keep in mind:

  1. If your opening and closing tags aren't on the same line, they should both be at the same level of indentation. (Hint, look at the vertical lines in VS Code that indicate which closing tag connects with which opening tag)
  2. If you need to indent more than one line at once, you can highlight multiple lines and press the 'Tab' key to indent them all simultaneously. "Un"-indenting multiple lines is the same, only instead of pressing just 'Tab', press and hold 'Shift' and then 'Tab'.

So yeah, take the time to indent your code! And the nice thing about VS Code (and most text editors)is that they will frequently indent your code for you automatically!

And some people have personal preferences for the particular way they like to indent some lines. That's fine, as long as it is easy to tell which elements are inside of which elements.
