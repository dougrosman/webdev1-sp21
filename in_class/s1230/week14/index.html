  1. Don't forget to create a <title> for your page (don't just leave it as 'Document')
  2. Change default styling for menu links (remove underlines, change from the default blue, and remove bullets from list items.
  3. Avoid using px whenever possible when setting sizes on things.
  4. Test your pages at multiple screen sizes.
  5. Avoid positioning elements on your page using px (and don't use absolute positioning to place things on your page (unless there are specific reasons to do so)).
  6. Avoid setting heights on containers (and widths, for that matter) -- use max-width or min-width instead!
  7. Avoid positioning things on your page using wild margins.
  8. If you're doing math, you're doing it wrong! (most of the time)
  9. Use 'relative' filepaths, not 'absolute' filepaths
  10. Don't start any links with '/' (it might work in VS Code/Live server, but not GitHub Pages.
  11. Resize your images before putting them in your repo.
